# Claude Task

Resolve `<PLUGIN_ROOT>` to the installed plugin root. Inspect the current permission profile first. In automatic-approval mode, invoke immediately without asking the user for pre-confirmation; when a workspace boundary is known to block required access, make the tool call with sandbox escalation so the host approval engine can decide it. In unrestricted/full-access mode, run directly without requesting escalation. In a workspace sandbox with approvals disabled, do not attempt an invocation that requires out-of-bound access; report that the permission profile must change. When sandbox escalation is available, request it only when Claude credentials, network access, or plugin state are outside the permitted boundary. Run `node "<PLUGIN_ROOT>/scripts/claude-companion.mjs" task "<task>"` from the target project root. Tasks are read-only by default and use the standard Sonnet profile; add `--write` only when the user explicitly authorizes edits. Use `--task-profile quick|standard|deep`; deep explicitly selects Opus and is high-cost. Pass `--model fable` unchanged when the user requests Fable. Add `--background` for detached execution and one of `--resume <session-id>`, `--continue`, or `--fresh` for session routing. Never resume, retry, select Opus/Fable, or add a fallback model automatically. Forward explicitly requested `--model`, `--effort`, `--task-profile`, `--max-turns`, `--max-budget-usd`, `--context`, `--finalize-at-turn`, or `--prompt-file` controls. Prefer `--prompt-file` for long user-provided inputs so shell and approval displays remain compact. Report the disclosure summary when material. Preserve the result, phase, requested and effective models, session resume hint, token count, cost, cumulative resume-chain cost, turn count, and duration. Treat `--max-budget-usd` as a soft target, not a hard billing ceiling. For a background launch, state that final usage metrics are available from `claude-result`, not from the initial job record. On failure, report the persisted job state, error kind, suggested action, stderr, and exit status; do not present a failed or cancelled job as completed. If host policy denies repository export, explain the boundary once and retry only after a permission-profile change or host-provided authorization change.
